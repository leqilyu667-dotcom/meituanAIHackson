# 三、模块一：数据小二（全部 P0 功能）
## 3.1 经营日报 - 日/周核心经营概览
**优先级**：P0
**功能定位**：门店经营数据一站式可视化看板，快速掌握日/周维度核心经营健康度。

### 业务流程
进入经营日报 → 切换「日报/周报」视图 → 查看指标卡片+环比数据 → 查看异常标红提醒
### 输入
视图类型（日报 / 周报）、门店ID
### 输出
1. 核心指标卡片：营收总额、客流量、客单价、订单完成率；
2. 环比对比数据：与上一周期（昨日/上周同日）对比，箭头+颜色区分升降；
3. 异常标红：指标波动超30%自动标红提示。

### 业务规则
1. 周报默认加载**过去7天汇总数据**，日报加载当日实时数据；
2. 数据来源：营收取自「营收记账模块」，客流、预约数据取自「客服小二-预约管理」；
3. 指标按全局口径计算，不可手动修改；
4. 页面采用卡片式布局。

### 异常处理
1. 数据源为空：展示「暂无数据」占位提示；
2. 加载超时：弹出「数据加载失败，请稍后重试」。

### 验收标准
1. 指标计算与上游数据源完全一致；
2. 环比、颜色/箭头标识正常，异常标红生效；
3. 视图切换无报错，加载流畅。

## 3.5 爆款趋势 - 站内--最热试戴标签识别
**优先级**：P0
**功能定位**：门店&平台款式热度智能分析，捕捉热门美甲特征趋势。适配真实美甲业务场景：大量纯色本甲、裸色基础款式无工艺无装饰，维度打标不强制、不补全，热度分按维度权重+行为数据加权计算。

### 业务流程
进入爆款报告 → 选择维度（门店/平台）→ 选择聚合粒度（一级/二级/三级）→ 查看标签热度榜单+趋势视图

### 输入
查询维度（门店 / 平台）、聚合粒度（一级/二级/三级）、时间范围（固定近7天）

### 输出
1. 当前粒度下热门标签组合TOP榜单（含完整度标识）；
2. 近7天标签热度变化趋势图；
3. 单标签明细：试戴量、订单量、综合热度分。

---

### 3.5.1 维度分级标准

| 层级 | 维度 | 是否强制 | 缺失处理 |
|------|------|----------|----------|
| 核心维度 | 甲型 | **必选** | 无，款式必须标注甲型 |
| 核心维度 | 色调 | **必选** | 无，款式必须标注色调 |
| 可选维度 | 工艺 | 非强制 | 允许为空，**不自动补默认值**，保留空值参与聚合 |
| 可选维度 | 装饰元素 | 非强制 | 允许为空，**不自动补默认值**，保留空值参与聚合 |
| 兜底维度 | 风格 | 非强制 | 缺失时**自动补「未标注风格」**，确保风格维度不缺失 |

**场景说明**：
- 纯色本甲、裸色基础款 → 工艺=空、装饰=空，不做任何强制补全
- 法式美甲、猫眼美甲 → 工艺有值、装饰可空可填
- 所有款式 → 风格缺失时统一补「未标注风格」

---

### 3.5.2 三层聚合粒度

#### 一级聚合（默认展示）
**粒度范围**：甲型 + 色调（仅核心维度）

适用场景：商家日常快速概览，数据集中，置信度高。纯色基础款与复杂款在核心维度层面公平比较。

聚合规则：
- 按「甲型+色调」组合汇总试戴量、订单量
- 工艺、装饰均不参与一级聚合
- 风格补「未标注风格」后参与标签展示，但不影响一级聚合的统计单元

#### 二级聚合（中粒度）
**粒度范围**：甲型 + 色调 + 工艺 + 装饰

适用场景：需要区分工艺复杂度、装饰富度的分析场景，如对比「杏仁甲+裸色+渐变+碎钻」与「杏仁甲+裸色++」（工艺空、装饰空）。

聚合规则：
- 工艺为空时保留空字符串参与聚合，不归一为「无工艺」
- 装饰为空时保留空字符串参与聚合，不归一为「无装饰」
- 展示时工艺/装饰为空显示为「—」

#### 三级聚合（全维度精细，仅后台用）
**粒度范围**：甲型 + 色调 + 工艺 + 装饰 + 风格（完整五维）

适用场景：后台详细数据导出、深度分析，前端不做默认展示。

### 3.5.3 热度分计算公式

#### 综合热度分

```
综合热度分 = 标签维度得分 × 行为数据得分

标签维度得分 = 甲型热度 × 0.30
             + 色调热度 × 0.30
             + 工艺热度 × 0.20
             + 装饰热度 × 0.15
             + 风格热度 × 0.05

行为数据得分 = 试戴量 × 0.6 + 订单量 × 0.4
```

#### 维度单项热度定义

| 维度 | 热度计算方式 | 数据来源 |
|------|-------------|----------|
| 甲型热度 | 该甲型在全量试戴中的占比 × 100 | AI试戴数据 |
| 色调热度 | 该色调在全量试戴中的占比 × 100 | AI试戴数据 |
| 工艺热度 | 该工艺在**有工艺标签的款式**中的占比 × 100 | AI试戴数据 |
| 装饰热度 | 该装饰在**有装饰标签的款式**中的占比 × 100 | AI试戴数据 |
| 风格热度 | 该风格在全量试戴中的占比 × 100 | AI试戴数据 |

#### 计分特殊规则

1. **无工艺款式**（工艺=空）：工艺热度按 **0** 计算，不参与工艺维度得分，权重自动向甲型、色调倾斜
2. **无装饰款式**（装饰=空）：装饰热度按 **0** 计算，不参与装饰维度得分
3. **未标注风格**：风格热度按 0 计算，权重 5% 实质上无影响
4. 维度得分为 0 时，不干扰其他维度的正常排序

> 设计意图：纯色基础款（无工艺、无装饰）在核心维度（甲型+色调）上依然能够公平参与热度竞争，不会因为缺失工艺/装饰标签而排名被稀释或榜单不可信。

#### 示例计算

款式 A：`杏仁甲 + 裸色 + 渐变 + 碎钻 + 温柔风`，试戴 520 次，订单 64 单
- 甲型热度 = 85（杏仁甲热度高）→ 85 × 0.30 = 25.5
- 色调热度 = 90（裸色热度高）→ 90 × 0.30 = 27.0
- 工艺热度 = 70（渐变热度）→ 70 × 0.20 = 14.0
- 装饰热度 = 60（碎钻热度）→ 60 × 0.15 = 9.0
- 风格热度 = 75（温柔风热度）→ 75 × 0.05 = 3.75
- 标签维度得分 = 25.5 + 27.0 + 14.0 + 9.0 + 3.75 = **79.25**
- 行为数据得分 = 520 × 0.6 + 64 × 0.4 = 312 + 25.6 = **337.6**
- **综合热度分 = 79.25 × 337.6 = 26754.8**

款式 B：`方圆甲 + 裸色 + [空] + [空] + 未标注风格`（纯色基础款），试戴 200 次，订单 30 单
- 甲型热度 = 78 → 78 × 0.30 = 23.4
- 色调热度 = 90 → 90 × 0.30 = 27.0
- 工艺热度 = **0**（无工艺）
- 装饰热度 = **0**（无装饰）
- 风格热度 = **0**（未标注风格）
- 标签维度得分 = 23.4 + 27.0 + 0 + 0 + 0 = **50.4**
- 行为数据得分 = 200 × 0.6 + 30 × 0.4 = 120 + 12 = **132.0**
- **综合热度分 = 50.4 × 132.0 = 6652.8**

> 纯色基础款虽然在工艺/装饰维度得分为 0，但在甲型+色调核心维度上正常得分，加行为数据加权后仍然能在同核心维度款式中公平排名。

1. OpenClaw 推送素材接口：`POST /api/common/openclaw/xhs/push`
2. 素材复审回调接口：`POST /api/merchant/material/review/callback`

2. 违规图片：直接拦截，不进入待审库；
