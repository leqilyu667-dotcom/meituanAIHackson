# 三、模块一：数据小二（全部 P0 功能）
## 3.1 经营日报 - 日/周核心经营概览
**优先级**：P0
**功能定位**：门店经营数据一站式可视化看板，快速掌握日/周维度核心经营健康度。

### 业务流程
进入经营日报 → 切换「日报/周报」视图 → 查看指标卡片+环比数据 → 查看异常标红提醒
### 输入
视图类型（日报 / 周报）、门店ID
### 输出
1. 核心指标卡片：营收总额、客流量、客单价、订单完成率；
2. 环比对比数据：与上一周期（昨日/上周同日）对比，箭头+颜色区分升降；
3. 异常标红：指标波动超30%自动标红提示。

### 业务规则
1. 周报默认加载**过去7天汇总数据**，日报加载当日实时数据；
2. 数据来源：营收取自「营收记账模块」，客流、预约数据取自「客服小二-预约管理」；
3. 指标按全局口径计算，不可手动修改；
4. 页面采用卡片式布局。

### 异常处理
1. 数据源为空：展示「暂无数据」占位提示；
2. 加载超时：弹出「数据加载失败，请稍后重试」。

### 验收标准
1. 指标计算与上游数据源完全一致；
2. 环比、颜色/箭头标识正常，异常标红生效；
3. 视图切换无报错，加载流畅。

## 3.2 经营日报 - 订单结构分析
**优先级**：P0
**功能定位**：联动爆款趋势、作品标签数据，展示款式维度经营贡献。

### 业务流程
打开经营日报 → 切换至订单结构分析标签页 → 查看标签营收榜、AI试戴引流数据
### 输入
视图类型（日报/周报）、门店ID
### 输出
1. 热门标签营收TOP榜：按**标签组合**展示营收占比、订单量；
2. AI试戴引流数据：试戴带来的订单总量、试戴转化率。

### 业务规则
1. 款式数据自动从「爆款趋势+作品标签」同步，无需人工录入；
2. 标签组合按综合营收排序，展示TOP10榜单；
3. 试戴转化率 = 试戴后成交订单数 ÷ 总试戴次数。

### 异常处理
无款式/试戴数据时，展示「暂无相关款式数据」。

### 验收标准
1. 榜单排序、营收占比、转化率计算准确；
2. 跨模块数据同步正常。

## 3.5 爆款趋势 - 站内--最热试戴标签识别
**优先级**：P0
**功能定位**：门店&平台款式热度智能分析，捕捉热门美甲特征趋势。适配真实美甲业务场景：纯色本甲、裸色基础款无工艺无装饰，风格由AI智能推荐一键确认，核心三维（甲型+色调+风格）必选，工艺/装饰可选。

### 业务流程
进入爆款报告 → 选择维度（门店/平台）→ 选择聚合粒度（一级/二级/三级）→ 查看标签热度榜单+趋势视图 → 缺失风格款式触发AI预标 → 商家一键确认

### 输入
查询维度（门店 / 平台）、聚合粒度（一级/二级/三级）、时间范围（固定近7天）

### 输出
1. 当前粒度下热门标签组合TOP榜单（含完整度标识）；
2. 近7天标签热度变化趋势图；
3. 单标签明细：试戴量、订单量、综合热度分；
4. 待标风格款式列表（仅当存在风格缺失数据时）。

---

### 3.5.1 维度分级标准

| 层级 | 维度 | 是否强制 | 缺失处理 |
|------|------|----------|----------|
| 核心维度 | 甲型 | **必选** | 无，款式必须标注甲型 |
| 核心维度 | 色调 | **必选** | 无，款式必须标注色调 |
| 核心维度 | 风格 | **必选** | 缺失时触发 **AI 智能推荐**，商家一键确认后补全，不允许长期空置 |
| 可选维度 | 工艺 | 非强制 | 允许为空，**不自动补默认值**，保留空值参与聚合 |
| 可选维度 | 装饰元素 | 非强制 | 允许为空，**不自动补默认值**，保留空值参与聚合 |

**场景说明**：
- 完整款（五维全有）→ 正常参与排序，无额外标识
- 基础款（工艺/装饰为空）→ 显示「基础款」灰色徽标，热度分由核心三维主导
- 新品入库（风格缺失）→ AI 根据「甲型+色调+工艺」自动推荐 Top 风格，商家点击确认，零负担完成必填

### 3.5.2 三层聚合粒度

#### 一级聚合（默认展示）
**粒度范围**：甲型 + 色调 + 风格（核心三维）

适用场景：商家日常快速概览，数据集中，置信度高。三位一体概括款式特征，风格作为区分「杏仁甲+裸色+法式」vs「杏仁甲+裸色+温柔风」的核心维度。

聚合规则：
- 按「甲型+色调+风格」组合汇总试戴量、订单量
- 工艺、装饰均不参与一级聚合
- 风格为款式必选维度，不出现空值

#### 二级聚合（中粒度）
**粒度范围**：甲型 + 色调 + 风格 + 工艺 + 装饰（全维度）

适用场景：需要区分工艺复杂度、装饰富度的分析场景。

聚合规则：
- 工艺为空时保留空字符串参与聚合，不归一为「无工艺」
- 装饰为空时保留空字符串参与聚合，不归一为「无装饰」
- 展示时工艺/装饰为空显示为「—」
- 工艺=空 或 装饰=空 → 行尾灰色「基础款」徽标

#### 三级聚合（仅后台用）
**粒度范围**：完整五维，按原始标签精确分组

适用场景：后台详细数据导出、深度分析，前端不做默认展示。

### 3.5.3 AI 智能风格推荐

款式入库/打标时，若「风格」字段缺失，系统自动触发 AI 推荐：

**推荐逻辑**：
1. 输入：甲型 + 色调 + 工艺（三项中已填写的值）
2. 匹配规则：查询全平台标注了该组合中同甲型、同色调的款式，统计最常关联的风格 Top 3
3. 工艺为空时仅用甲型+色调匹配，确保基础款也能准确推荐
4. 输出：Top 3 推荐风格 + 置信度百分比

**前端交互**：
- 页面顶部出现「待标注风格」提醒条：`有 N 个新款式未标注风格，点击处理`
- 展开推荐列表：每条款式展示缩略图 + 甲型/色调/工艺 + Top 3 推荐按钮
- 商家点击任一推荐按钮 → 即时确认，无需额外操作
- 支持「全部采用首选」一键批量确认

**示例**：
- 款式 A：杏仁甲 + 裸色 + 渐变 → AI 推荐「温柔风 78%」「简约风 12%」「日式 6%」
- 款式 B：方圆甲 + 红色系 + 纯色 → AI 推荐「法式 65%」「简约风 22%」「温柔风 8%」

### 3.5.4 热度分计算公式

#### 综合热度分

```
综合热度分 = 标签维度得分 × 行为数据得分

标签维度得分 = 甲型热度 × 0.30
             + 色调热度 × 0.30
             + 风格热度 × 0.10
             + 工艺热度 × 0.15
             + 装饰热度 × 0.15

行为数据得分 = 试戴量 × 0.6 + 订单量 × 0.4
```

#### 维度单项热度定义

| 维度 | 热度计算方式 | 数据来源 |
|------|-------------|----------|
| 甲型热度 | 该甲型在全量试戴中的占比 × 100 | AI试戴数据 |
| 色调热度 | 该色调在全量试戴中的占比 × 100 | AI试戴数据 |
| 风格热度 | 该风格在全量试戴中的占比 × 100 | AI试戴数据 |
| 工艺热度 | 该工艺在**有工艺标签的款式**中的占比 × 100 | AI试戴数据 |
| 装饰热度 | 该装饰在**有装饰标签的款式**中的占比 × 100 | AI试戴数据 |

#### 计分特殊规则

1. **无工艺款式**（工艺=空）：工艺热度按 **0** 计算
2. **无装饰款式**（装饰=空）：装饰热度按 **0** 计算
3. 核心三维（甲型+色调+风格）必有值，各维度权重恒定，不因可选维度缺失而重新分配
4. 维度得分为 0 时，不干扰其他维度的正常排序

> 设计意图：风格升级为必选后权重提升至 10%，核心三维合计 70% 主导热度排序。纯色基础款（无工艺、无装饰）在核心三维上依然能够公平参与热度竞争。

#### 示例计算

款式 A：`杏仁甲 + 裸色 + 温柔风 + 渐变 + 碎钻`（完整款），试戴 520 次，订单 64 单
- 甲型热度 = 85 → 85 × 0.30 = 25.5
- 色调热度 = 90 → 90 × 0.30 = 27.0
- 风格热度 = 75 → 75 × 0.10 = 7.5
- 工艺热度 = 70 → 70 × 0.15 = 10.5
- 装饰热度 = 60 → 60 × 0.15 = 9.0
- 标签维度得分 = 25.5 + 27.0 + 7.5 + 10.5 + 9.0 = **79.5**
- 行为数据得分 = 520 × 0.6 + 64 × 0.4 = 312 + 25.6 = **337.6**
- **综合热度分 = 79.5 × 337.6 = 26839**

款式 B：`方圆甲 + 红色系 + 法式 + [空] + [空]`（纯色基础款），试戴 200 次，订单 30 单
- 甲型热度 = 78 → 78 × 0.30 = 23.4
- 色调热度 = 82 → 82 × 0.30 = 24.6
- 风格热度 = 72 → 72 × 0.10 = 7.2
- 工艺热度 = **0**（无工艺）
- 装饰热度 = **0**（无装饰）
- 标签维度得分 = 23.4 + 24.6 + 7.2 + 0 + 0 = **55.2**
- 行为数据得分 = 200 × 0.6 + 30 × 0.4 = 120 + 12 = **132.0**
- **综合热度分 = 55.2 × 132.0 = 7286**

分页款式列表、单款基础信息、款式状态标识

### 核心字段
款式ID、门店ID、款式名称、封面图地址、款式状态、创建时间、最后操作人

### 业务规则
1. 列表默认按**创建时间倒序**展示，固定单一排序规则，本期不支持切换排序维度；
2. 仅提供状态筛选：全部 / 已上架 / 未上架，无其他筛选条件；
3. 列表分页加载，保证页面浏览流畅；
4. AI 爆款素材生成的新款式自动进入列表，默认状态为「未上架」；
5. 权限区分：商家主账号、门店员工账号均可正常查看列表。

### 异常处理
1. 暂无任何款式：页面展示「暂无相关款式」占位提示；
2. 数据加载超时：弹出「数据加载失败，请稍后重试」；
3. 筛选参数异常：自动重置为「全部」状态，返回全量列表。

### 验收标准
1. 全量款式正常分页加载，默认排序规则生效；
2. 状态筛选功能正常，可准确区分已上架/已下架款式；
3. 页面UI、组件样式与 `customer` 分支设计保持统一；
4. 列表可正常承接后续上下架、编辑、查看详情等操作。

## 6.2 款式上下架管理 - 单条/批量操作
**优先级**：P0
**功能定位**：商家自主控制美甲款式在C端的展示状态，支持单条、批量上下架，与系统智能货架优化形成双向联动。
> 备注：定时上下架、分时段上架等P1功能本期不开发。

### 业务流程
1. 单条操作：在款式列表点击「上架/下架」按钮 → 二次确认（可选）→ 状态变更 → 同步全关联模块
2. 批量操作：勾选多条款式 → 点击批量上架/批量下架 → 确认操作 → 批量变更状态

### 输入
款式ID（单条/批量集合）、操作类型（上架/下架）、门店ID

### 输出
款式最新状态、操作结果提示、操作日志

### 核心字段
款式ID、门店ID、款式状态、操作类型、操作人ID、操作时间、操作备注

### 状态流转规则（全局枚举，与 `customer` 分支对齐）
```
已下架 → 已上架
已上架 → 已下架
```

### 业务规则
1. 款式**下架仅隐藏展示，不物理删除**，历史订单、统计数据永久保留；
2. 执行上架操作时，强制校验必填标签：甲型、色调、风格不能为空，校验失败则禁止上架并弹窗提示；
3. 上下架状态实时同步至C端用户页面、AI试戴素材池、货架优化模块；
4. 系统智能调整产生的上下架动作，在列表中标注「系统调整」标识，支持商家一键恢复手动状态；
5. 所有上下架操作强制记录日志，日志保存时长 ≥ 180 天，支持溯源查询；
6. 批量操作完成后，页面展示本次操作成功款式数量。

### 异常处理
1. 款式必填标签缺失：弹窗提示「请完善甲型/色调/风格标签后再上架」，终止操作；
2. 批量选中无有效款式：提示「未选中任何款式，请重新选择」；
3. 状态同步失败：记录异常日志，页面提示「状态变更成功，数据同步异常，请稍后检查」。

### 验收标准
1. 单条、批量上下架状态流转正常，枚举值与 `customer` 分支完全一致；
2. 上架标签校验规则生效，缺失必填项无法完成上架；
3. 状态变更后，C端、AI试戴池、货架模块同步生效；
4. 操作日志完整记录，标识区分「手动操作/系统调整」。

## 6.3 款式信息编辑

### 业务流程

### 输入

### 输出
更新后的款式全量信息、操作结果提示、操作日志

### 核心字段
款式ID、门店ID、款式名称、售价、展示描述、甲型、色调、风格、工艺、装饰、封面图、细节图、最后编辑人、编辑时间

### 业务规则
1. 甲型、色调、风格为**必填项**，保存时强制校验，不能为空；工艺、装饰为可选项，允许留空；
2. 图片遵循全局规范：格式 JPG/PNG，单张大小 ≤ 10MB，系统自动裁剪、隐私脱敏、去除水印；
3. 标签、素材、基础信息修改后，实时同步至爆款趋势、订单结构分析、C端展示、AI试戴模块；
4. 编辑操作全程记录日志，保存时长 ≥ 180 天；

### 异常处理
1. 必填标签为空：弹窗提示「甲型/色调/风格为必填项，请补充完整」，禁止保存；
2. 图片格式/大小不符合规范：提示「图片格式或大小超出限制，请重新上传」；
3. 内容输入非法字符：拦截提交，提示「存在非法字符，请修改后重试」。

### 验收标准
1. 编辑弹窗/页面正常打开，字段可正常录入、修改；
2. 必填项校验、图片校验规则生效；
3. 保存后信息实时更新，全关联模块数据同步无误；
4. 编辑日志完整可查。

# 七、模块联动规则（商品管理 全局补充）
## 7.1 跨模块联动（强制生效）
1. **与货架优化联动**
   货架优化的智能上下架、智能排序，同步在商品列表标注系统标识；商家手动操作优先级高于系统智能调整。
2. **与爆款素材生成联动**
   AI生成的爆款素材自动新增为待上架款式，进入商品列表，支持直接编辑、上架。
3. **与数据小二（经营日报/爆款趋势）联动**
   款式标签、订单、营收、试戴数据双向实时同步，保证全模块数据统一。
4. **与运营建议联动**
   运营建议内「补充款式、调整推荐」类入口，可直接跳转至商品管理对应款式页面。
5. **与客户分支（customer）联动**
   款式状态、素材、排序、标签修改后，实时同步至C端页面、AI试戴素材池。

## 7.2 权限总规则
1. 商家主账号：拥有商品管理**全功能权限**（查看、编辑、上下架、排序、查看数据看板）；
2. 门店员工账号：仅可查看款式列表、数据看板，禁止编辑、上下架、排序操作。

## 7.3 本期统一禁止开发（P1及以上）
1. 款式物理删除、库存管理、多规格、套餐组合；
2. 定时上下架、分组管理、素材批量替换；
3. 标签筛选、关键词搜索、多维度列表排序；
4. 数据导出、自定义统计时间、单品竞品对比；
5. 款式审核流程、多门店商品同步功能。
